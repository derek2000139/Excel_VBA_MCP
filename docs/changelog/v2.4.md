# v2.4 变更日志

**版本**：`v2.4.0`\
**发布日期**：`2026-03-30`\
**相比上一版本**：v2.3.x

***

## 1. 版本概述

v2.4 是 V2 阶段的重大扩充版本，主要完成了以下工作：

- 新增 **table** 域（8工具）：表格对象管理
- 新增 **analysis** 域（6工具）：结构扫描、公式分析、链接审计
- 新增 **workbook_ops** 域（6工具）：工作簿治理与导出
- 扩充 **sheet** 域（+4工具）：copy/move/hide/unhide
- 扩充 **range** 域（+2工具）：find_replace/autofit
- 新增 **Tool Manifest 扩展字段**
- 新增 **server.health backend 结构**
- 完成 Profile 重整与工具域梳理

***

## 3. Bundle 变更

### 3.1 新增 Bundle

| Bundle | 工具数 | 依赖 | 说明 |
|--------|---:|------|------|
| data | 8 | foundation | table 表格管理 |
| analysis | 6 | foundation | 分析工具集 |
| workbook_ops | 6 | foundation | 工作簿治理与导出 |
| edit_structure | 6 | foundation | sheet.copy/move/hide/unhide + range.find_replace/autofit |

### 3.2 Bundle 调整

- `data` bundle：从 analysis 中拆分 table 工具，独立为 data bundle
- `analysis` bundle：扩充至 6 工具（包含 audit.list_operations）
- `edit_structure` bundle：从 edit_basic 中拆分结构编辑工具

## 5. 框架预留

### 5.1 Tool Manifest 新增字段

- `execution_mode`：执行模式（sync/async/batch）
- `backend_requirement`：后端要求（excel/python/any）
- `supports_batch`：是否支持批量操作

### 5.2 server.health 新增结构

```json
{
  "backend": {
    "version": "2.4.0",
    "supports_batch": false,
    "supports_artifact_export": false,
    "execution_mode": "excel"
  }
}
```

### 5.3 bundles.yaml 预留

- `artifact_export` bundle（待实现）

## 6. 重大修复

### 6.1 Host 诊断参数

新增以下诊断参数：

- `--dump-tools`：输出 Profile 所有工具列表
- `--dump-tools-with-index`：输出带索引的工具列表
- `--dump-profile-resolution`：输出 Profile 解析过程

### 6.2 启动预算警告

当 Profile 工具数超过 `tool_budget` 时，启动时输出警告到 stderr。

## 8. 升级注意事项

### 8.1 工具名变化

新增工具均为全新工具，不影响已有工具名。

### 8.2 Profile 兼容

已有 Profile 配置可继续使用，但工具数增加可能导致 Trae AI 等客户端截断。建议切换到 `data_workflow` 或减少 `--enable-bundle`。

### 8.3 依赖变化

无新增外部依赖。
